Installation

There are two ways Soho can be installed. For existing sites that already have content, the first method will only involve the installation and activation of the theme. The second option includes a full Drupal 7 install with the same SQL database file used to create the theme demo. Please choose which installation method works best for you and follow the instructions. We do highly recommend you perform the full install if you are new to Drupal or don't have any existing content you need to work with.

Theme Only

Option 1:
  • Visit yourdomain.com/admin/appearance/install
  • Upload the Soho.zip file found inside the /theme folder in the download files
  • Enable and set to default
Option 2:
  • Extract the Soho.zip file and upload the Soho folder via FTP to /sites/all/themes
  • Visit yourdomain.com/admin/appearance/
  • Scroll down to "DISABLED THEMES" and set and enable Soho

Full Drupal Install

The following steps are designed for people that have experience manually installing Drupal and importing databases. If you do not feel comfortable performing these steps yourself, contact your server administrator for help. NOTE this option is for a completely new Drupal install. Do not import the provided SQL file into an existing Drupal database or you will lose all of your data!

  1. Create a database for this installation of Drupal and import the SQL file that came with the theme.
  2. Follow these directions to create the settings.php file. Save this file somewhere so you can copy it over to the Drupal installation we have provided.
  3. Copy the entire contents of the /demo/soho folder from the download files to the location on your web server where you want your site to be accessed. If you want your primary domain to point to your install of Soho, copy everything to the root folder. If you want your site to be accessed as a subsection of your domain (for example: yourdomain.com/drupal) then simply rename the /demo/soho folder to the desired name and upload to the root of your server. Also be sure to copy over the settings.php file mentioned in the previous step
  4. Login to your site with the username admin and the password admin and visit /admin/people to change the admin username and password.
  5. Go through the Drupal configuration found in /admin/config options and update all the necessary information to reflect your website. Also be sure to visit /admin/config/media/file-system and change the Temporary Directory if needed.

Soho Sub-Theme

Soho comes with a shell sub-theme called soho_sub that you can use to make customizations to the style and template structure of the parent theme without having to modify the parent theme itself. This is useful if you plan on updating the parent theme as we release updates on ThemeForest. If you performed the full install you will already see soho_sub in the list of themes available. If you did the standalone install you can find the soho_sub.zip file in the root of the download files and you can install it like any other theme.

There is an empty stylesheet already defined which is /css/custom.css (inside the soho_sub folder). Use this file to add all of the custom CSS that you plan on using. You can also copy any template file from the /soho/tpl folder into the /soho_sub folder and it will listen to the sub-themes template file first. This will allow you to update the parent theme as we make updates without losing your customizations. Just pay attention to the changelog and compare the parent theme files with yours and make any adjustments as needed.

You will need to configure the sub-themes theme settings separately from the parent theme's, so visit /admin/appearance/settings/soho_sub to configure the theme settings for the sub-theme. You also need to configure the block regions separately.

Soho Utilities

This is the core helper module for the Soho theme and is required by the Slider, Gallery and Striped modules. It has no settings and needs no additional configuration.

Soho Slider



This module creates a custom entity called Soho Slider which can be used to create a slider like the one seen on the following page. Once installed, you can navigate to /admin/structure/soho-slider or simply click the Structure link in your admin menu and click Soho Sliders. Once there click the "Add Soho slider" link and you can begin creating your slider. Enter a name for your slider and then proceed to use the following fields:

Slider Content



This field accepts a slide title, caption and image. You can add additional slides by pressing the "add another item" button. There is no limit to how many slides the slider can use.

Slider Icons



This optional field allows you to add social icons and links which will open in an overlay over the slider when the appropriate button is clicked in the slider controls. Simply select the icon you want from the dropdown and enter your link.

Slider Settings



Finally you can configure the overall settings for the slider. You can adjust the time between the slide animation, the effect, autoplay, enable/disable the controls and the social icon button.

Slider Block

The Soho Slider module automatically creates a block for every slider you create. Visit /admin/structure/block and add your slider block(s) to the region of your choice. All Soho Sliders will be labeled Soho Slider: Your Title and you can use the block visibility settings to dictate what nodes you want the slider to be visible on.

Soho Gallery



This module creates a custom entity called Soho Gallery which can be used to create a gallery like the one seen on the following page. Once installed, you can navigate to /admin/structure/soho-gallery or simply click the Structure link in your admin menu and click Soho Galleries. Once there click the "Add Soho gallery" link and you can begin creating your gallery. Enter a name for your gallery and then proceed to use the following fields:

Gallery Content



This field accepts a gallery title and image. You can add additional items by pressing the "add another item" button. There is no limit to how many items the gallery can use.

Gallery Settings



Here you can select the layout you want to use for your gallery. You have six options to choose from:

  • Kenburns - http://refaktorthemes.com/soho/gallery-kenburns
  • Ribbon - http://refaktorthemes.com/soho/gallery-ribbon
  • Grid - http://refaktorthemes.com/soho/gallery-grid
  • Grid 2 - http://refaktorthemes.com/soho/gallery-grid-margin
  • Masonry - http://refaktorthemes.com/Soho/gallery-masonry
  • Masonry - http://refaktorthemes.com/soho/gallery-masonry-margin

Gallery Block

The Soho Gallery module automatically creates a block for every gallery you create. Visit /admin/structure/block and add your gallery block(s) to the region of your choice. All Soho Galleries will be labeled Soho Gallery: Your Title and you can use the block visibility settings to dictate what nodes you want the gallery to be visible on.

Soho Striped



This module creates a custom entity called Soho Striped which can be used to create a Striped block like the one seen on the following page. Once installed, you can navigate to /admin/structure/Soho-striped or simply click the Structure link in your admin menu and click Soho Striped. Once there click the "Add Soho striped" link and you can begin creating your striped block. Enter a name for your item and then proceed to use the following fields:

Striped Content



This field accepts a title, caption, image (within the content), background image, url, color and opacity. You can add additional items by pressing the "add another item" button. There is no limit to how many items the striped block can use but we recommend a maximum of 6.

Striped Settings



You can use these settings to configure the type of display, whether or not to display the header, and a global landing image and url.

Striped Block

The Soho Striped module automatically creates a block for every item you create. Visit /admin/structure/block and add your block(s) to the region of your choice. All Soho Striped blocks will be labeled Soho Striped: Your Title and you can use the block visibility settings to dictate what nodes you want the block(s) to be visible on.

Soho Features

If you used the "theme only" option to install the theme on an existing install that already has content we recommend you use our Soho Features module to create the additional content types, fields and Views needed to fully utilize the theme. You will find it in the /assets folder in the download files. This module is a custom Features module and will not run without it. We recommend copying all of the modules from the /modules folder into your install if you do not already have them. After you have all the modules simply upload and activate the Soho Features module and the content types, fields and views will automatically be created.

Menu

Soho has a block region called Header Menu which can be used to add any menu block you like. The demo simply uses the default Main Menu block but you can create a custom menu block if you like.



Note: make sure to check the "show as expanded" box on all parent items with children to ensure the dropdown menu works correctly.

Soho Views



An important part of Soho are the custom views that have been created to display Portfolio and Article content. To assist with that part of the Soho Utilities module is a custom views style plugin called Soho Views. With the Soho Utilities module enabled, a view format called Soho Views becomes available. This format contains various options which correlate to the different view displays available.

If you're not working off the demo content, we highly recommend you either use the included Features module or the views export code (found in /assets/views in the download files) to import the views as they were configured in the demo.

Content Types

Article

Soho uses the default Article content type to handle the blog content with a few additional fields. Please continue reading for an overview of the fields, templates and views used for articles in Soho.



Fields:

  • Title: The default Drupal title field.
  • Article Layout: List text field containing two Article layout options (simple and fullwidth). See below for the key/label list.
    • simple|Simple
fullwidth|Fullwidth
  • Featured Image: Optional image field that can be used to display an alternate image in a blog view.
  • Image: The default Drupal image field. Multiple images can be uploaded and will be presented in a slider.
  • Media Embed: Long text field that can be used to embed content such as a YouTube or Vimeo video. This will display in place of an image.
  • Article Category: An optional taxonomy term reference field.
  • Body: The default Drupal body field displayed below the image/media.
  • Tags: The default Drupal tags field.
  • Like: A field generated by the Soho Utilities module that can be used to add a "like" button to your articles.

Template files:

  • /tpl/node/node--article.tpl.php
  • /inc/article.inc

Views:

The demo uses two different blog layouts: fullscreen masonry and standard. If you did a theme only install you can use either the Soho Features module or the views export code found in /assets/views in the ThemeForest downloads file to quickly create these views. The standard with left/right sidebar displays article teasers but the fullwidth masonry is a collection of fields that uses a field rewrite option to display the layout.

Portfolio

Soho uses a custom content type called Portfolio to display the various Portfolio pages seen on the demo. Please continue reading for an overview of the fields, templates and views used for Portfolios in Soho.



Fields:

  • Title: The default Drupal title field.
  • Portfolio Layout: A select field with several options to select the type of portfolio layout for the node. See below for the key/label list.
    • fullscreen|Fullscreen
fullwidth|Fullwidth
ribbon|Ribbon
flow|Flow
simple|Simple
  • Image: The default Drupal image field. Multiple images can be uploaded and will be presented in a simple slider when using the "simple" layout option.
  • Media Embed: Long text field that can be used to embed content such as a YouTube or Vimeo video. This will display in place of an image.
  • Portfolio Gallery: Image field that accepts multiple uploads. This is used with all the layouts other than simple.
  • Body: The default Drupal body field displayed below the image/media.
  • Portfolio Category: An optional taxonomy term reference field which can be used to create an Isotope portfolio layout.
  • Like: A field generated by the Soho Utilities module that can be used to add a "like" button to your articles.
  • Hide Media: You can use this to hide the image/media field in the single node display. This is useful if you have an image/media item you're only using within a view display, but don't want to use it in the single node.

Template files:

  • /tpl/node/node--portfolio.tpl.php
  • /inc/portfolio.inc

Views:

The demo uses several different portfolio layouts which are provided by the Soho Utilities module. If you did a theme only install you can use either the Soho Features module or the views export code found in /assets/views in the ThemeForest downloads file to quickly create the Portfolio view and corresponding displays.

Soho Page

Soho uses a custom content type called Soho Page to display either a fullscreen image or video. Continue reading below for a rundown of the fields and templates used with this content type:



Fields:

  • Title: The default Drupal title field.
  • Soho Page Image: Image field that allows for one image.
  • Soho Page Video: Long text field to embed something like a YouTube video.

Template files:

  • /tpl/node/node--soho-page.tpl.php

Landing Page

Soho uses a custom content type called Landing Page to display a page similar to the demo homepage. It has a field for a background image, a main logo image, a link, and an option to disable the page header. Continue reading below for a rundown of the fields and templates used with this content type:



Fields:

  • Title: The default Drupal title field.
  • LP Background Image: Image field for uploading a page background image.
  • LP Logo Image: Image field for uploading a centered logo image.
  • LP Logo Link: Text field for entering a URL to link from the logo image.
  • LP Disable Header: Boolean field to toggle the displaying of the page header.

Template files:

  • /tpl/node/node--landing-page.tpl.php

Contact

Soho uses a custom content type called Contact to display a page similar to the demo contact page. Continue reading below for a rundown of the fields and templates used with this content type:



Fields:

  • Title: The default Drupal title field.
  • Body: Standard body field, displays before the contact form and contact form right area.
  • Contact Form Right: Displays beside the contact form.
  • Map Coordinates: Text field for adding your location longitude and latitude. Example: 40.7236448, -74.0006793.

Template files:

  • /tpl/node/node--contact.tpl.php

Theme Settings



The Soho theme settings can be accessed via /admin/appearance/settings/ and then clicking on the name of your active theme. There are four sections, including the standard Drupal settings. Go through each tab and configure the settings to your liking.

Block Regions

Soho features 14 block regions where you can add your blocks. Visit yourdomain.com/admin/structure/block to add your own content to the defined block regions. Press "Add Block" to create a new custom blog, or select an existing block from the Disabled list below the defined block regions.
